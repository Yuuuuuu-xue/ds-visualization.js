Tech stack:

This library is mostly written in TypeScript with Less as the stylesheet.
This library did not use Canvas API or SVG elements. Instead, it modifies the DOM directly for creating graphs.

About this release:

Most of the functionalities are already implemented. On the next release, it will mostly be focused on styles (i.e. different types of vertex and overall styles) and provide more user-friendly endpoints for the user creating graphs.

Goal:

Its goal is to help developers to draw and visualize existing data structures, mainly graphs and trees (since a tree is also a type of a graph).

Usecase:

  • For the website that is similar to LeetCode that provides data structures related coding problems. The web app can use this library to draw graphs as a supplementary to explain a question or a solution.
  • lt can be used for web apps that want to visualize the map. The edge could be a street and the intersection will be a street intersection.
  • The library could also be used in social networks web apps such as Facebook or LinkedIn. It provides functionalities for end-users to visualize their networks where the vertex represents the user and an edge between two vertices represents two users are connected. Thus the user can view their connected users and their connected usersâ€™ network. And the different stages could be the network for each month so users can visualize how their network is changing and growing for a period of time.
